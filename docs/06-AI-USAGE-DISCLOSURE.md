# AI Usage Disclosure

## 1. Overview

AI was used as a development assistant during the planning and documentation phase of  Casino.

The goal was to apply a light SDD workflow and use AI to clarify requirements, explore design decisions, break down implementation tasks, and prepare documentation.

## 2. How AI Was Used

AI was used for:

- requirements analysis;
- interpreting the assessment requirements;
- defining product scope;
- identifying MVP boundaries;
- technical architecture discussion;
- database design planning;
- API contract planning;
- frontend/backend task breakdown;
- security checklist planning;
- testing strategy planning;
- documentation drafting;
- technical decision recording.

## 3. How AI Was Not Used

AI is not treated as the final authority.

The developer remains responsible for:

- final implementation;
- code review;
- debugging;
- security validation;
- testing;
- deployment;
- confirming that the application meets the assessment requirements.

## 4. Development Workflow

The project follows a light AI-assisted SDD workflow:

1. clarify product requirements;
2. define architecture;
3. define database model;
4. define API contracts;
5. split implementation into tasks;
6. document technical decisions;
7. implement and validate task by task.

## 5. Benefits

Using AI helped to:

- reduce ambiguity;
- make decisions explicit;
- keep the scope controlled;
- identify edge cases;
- prepare clearer documentation;
- produce an implementation plan that can be followed by a developer or coding agent.

## 6. Limitations

AI-generated documentation and suggestions require human validation.

The final repository must be reviewed and tested manually by the developer before submission.

## 7. Final Responsibility

All final implementation decisions, code quality, test execution, and deployment validation remain the developer's responsibility.
